# 阶段 7 / 任务 07：队列缓冲、配置、测试与验收

## 目标

定义 ACS 实时事件与离线上传事件的队列缓冲策略、阶段 7 配置校验、自动化测试、真实设备联调和阶段通过标准。

## 队列缓冲策略

阶段 7 不做平台主动历史查询，也不存在历史补偿期间的专用实时缓冲。实时 ACS 事件和设备恢复后上传的离线 ACS 事件统一进入 `FaceEventIngestionService` 队列，后台消费者按接收顺序处理。

| 状态 | 行为 |
| --- | --- |
| 队列正常 | 实时事件和离线上传事件直接入队。 |
| 队列接近满 | 记录 `Warn`，继续投递。 |
| 队列已满 | 按配置丢弃或降级，并记录 `Error`。 |
| 服务停止 | 尽量处理已入队事件；未入队事件不持久化。 |

阶段 7 不新增事件暂存表。服务崩溃或队列溢出时，已进入设备离线缓存但尚未上传的事件仍依赖设备后续上传能力；已被服务端接收但未入队的事件无法保证恢复。

## 缓冲容量

如不新增配置，采用内部默认值：

| 项目 | 默认建议 |
| --- | --- |
| 全局队列容量 | 2000 条。 |
| 队列接近满阈值 | 80%。 |
| 超限策略 | 记录 `Error` 并丢弃新事件；如实现配置项，应保持默认行为明确。 |

如果新增配置项，必须同步主方案和配置模板。

## 配置校验

沿用 `FaceEventLogging`：

| 配置项 | 默认值 | 校验 |
| --- | --- | --- |
| `Enabled` | `true` | false 时不启动阶段 7 模块。 |
| `SnapshotRootDirectory` | `snapshots` | 不存在则创建，创建失败则模块不可用。 |
| `ExcludedDeviceIds` | `[]` | 只影响阶段 7 事件入库与离线上传补偿。 |
| `ExcludedDeviceIps` | `[]` | 只影响阶段 7 事件入库与离线上传补偿。 |
| `OfflineCompensationEnabled` | `true` | false 时忽略 `byCurrentEvent == 2` 的离线事件，普通实时事件仍处理。 |
| `AlarmDeployType` | `0` | 阶段 7 固定客户端布防；非法值回退为 `0` 或配置校验失败。 |
| `QueueCapacity` | `2000` | 小于最小值时使用默认值。 |

排除设备不影响设备连接、权限同步、设备管理，也不影响后续 AIOP 联动配置引用。

## Mock SDK 集成测试

| 场景 | 预期 |
| --- | --- |
| 实时 ACS 成功事件 | 写入 `attendance_gate_v2`，保存图片。 |
| 实时 ACS 无图片 | 写入事件，`snapshot_path` 为空。 |
| 非 ACS 回调 | 不入库。 |
| 重复实时事件 | 第二次跳过。 |
| 离线 ACS 上传事件 | `byCurrentEvent == 2` 写入 `attendance_gate_v2`，raw payload 标记 `OfflineUpload`。 |
| 离线补偿关闭 | `OfflineCompensationEnabled = false` 时忽略 `byCurrentEvent == 2`。 |
| 重复离线事件 | 第二次跳过。 |
| 排除设备 | 不入库、不补偿。 |
| 主动查询禁用 | 不调用 `NET_DVR_GET_ACS_EVENT`，不读写 `face_event_checkpoint`。 |

## 真实设备联调

真实设备联调必须显式开启，不纳入默认自动化测试。

| 联调项 | 操作 | 通过标准 |
| --- | --- | --- |
| ACS 实时回调 | 现场触发人脸识别或门禁认证。 | 收到 `COMM_ALARM_ACS`，队列有事件。 |
| 实时入库 | 触发一次有效认证。 | `attendance_gate_v2` 新增一条记录。 |
| 抓拍保存 | 触发带图片事件。 | `snapshots` 下生成图片，表中有路径。 |
| 重复保护 | 重放同一 mock 事件或设备重复上传同一离线事件。 | 不重复入库。 |
| 离线事件上传补偿 | 保持客户端布防，断开服务或网络期间触发事件，恢复后观察设备上传。 | 收到 `byCurrentEvent == 2` 的 `COMM_ALARM_ACS` 并入库。 |
| V50 连续回调 | 连续触发多次。 | 均能接收；如不能，启用 V31 兼容适配。 |

## 阶段 7 通过标准

| 标准 | 说明 |
| --- | --- |
| 实时链路完整 | ACS 回调到数据库入库端到端可验证。 |
| 离线上传链路完整 | 设备恢复后上传的离线事件可识别、入队并入库。 |
| 主动查询未启用 | 阶段 7 不调用历史 ACS 查询，不维护 checkpoint。 |
| 重复保护稳定 | 实时重复、离线上传重复均不重复写表。 |
| 图片策略清楚 | 成功保存、失败降级、路径长度都有处理。 |
| 回调线程安全 | 不在 SDK 回调线程写库写文件。 |
| 排除设备生效 | 配置排除只影响阶段 7 事件能力。 |
| 数据库零结构变更 | 不修改现有表结构，不新增阶段 7 表。 |
