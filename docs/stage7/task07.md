# 阶段 7 / 任务 07：实时缓冲、配置、测试与验收

## 目标

定义历史补偿期间的实时事件缓冲策略、阶段 7 配置校验、自动化测试、真实设备联调和阶段通过标准。

## 实时缓冲策略

同一设备历史补偿进行中时，实时 ACS 事件先进入内存缓冲，避免历史补偿和实时入库交错导致断点语义混乱。

| 状态 | 行为 |
| --- | --- |
| 未补偿中 | 实时事件直接处理。 |
| 补偿中 | 实时事件进入该设备缓冲。 |
| 补偿成功 | 按接收顺序释放缓冲事件。 |
| 补偿失败 | 保留缓冲到短时间上限；超过上限后按实时事件降级处理并记录警告。 |
| 服务停止 | 尽量处理已入队事件；未处理内存缓冲不持久化。 |

阶段 7 不新增实时缓冲表，因此服务崩溃时内存缓冲可能丢失。现场应通过历史补偿再次覆盖缺口。

## 缓冲容量

如不新增配置，采用内部默认值：

| 项目 | 默认建议 |
| --- | --- |
| 单设备缓冲上限 | 1000 条。 |
| 缓冲最长保留 | 5 分钟。 |
| 超限策略 | 记录 `Error` 并按实时事件直接处理或丢弃，实施时按现场风险选择，默认直接处理。 |

如果新增配置项，必须同步主方案和配置模板。

## 配置校验

沿用 `FaceEventLogging`：

| 配置项 | 默认值 | 校验 |
| --- | --- | --- |
| `Enabled` | `true` | false 时不启动阶段 7 模块。 |
| `SnapshotRootDirectory` | `snapshots` | 不存在则创建，创建失败则模块不可用。 |
| `ExcludedDeviceIds` | `[]` | 只影响事件入库和历史补偿。 |
| `ExcludedDeviceIps` | `[]` | 只影响事件入库和历史补偿。 |
| `HistoryCompensationEnabled` | `true` | false 时只处理实时事件。 |
| `HistoryBatchSize` | `100` | 小于 1 使用默认值。 |

排除设备不影响设备连接、权限同步、设备管理，也不影响后续 AIOP 联动配置引用。

## Mock SDK 集成测试

| 场景 | 预期 |

## 真实设备联调

真实设备联调必须显式开启，不纳入默认自动化测试。

| 联调项 | 操作 | 通过标准 |
| --- | --- | --- |
| ACS 实时回调 | 现场触发人脸识别或门禁认证。 | 收到 `COMM_ALARM_ACS`，队列有事件。 |
| 实时入库 | 触发一次有效认证。 | `attendance_gate_v2` 新增一条记录。 |
| 抓拍保存 | 触发带图片事件。 | `snapshots` 下生成图片，表中有路径。 |
| 重复保护 | 重放同一 mock 事件或历史补偿覆盖实时事件。 | 不重复入库。 |
| 历史补偿 | 设备离线期间触发事件后恢复。 | 补偿后 checkpoint 前移。 |
| V50 连续回调 | 连续触发多次。 | 均能接收；如不能，启用 V31 兼容适配。 |

## 阶段 7 通过标准

| 标准 | 说明 |
| --- | --- |
| 实时链路完整 | ACS 回调到数据库入库端到端可验证。 |
| 历史链路完整 | checkpoint 查询、历史事件查询、入库、前移可验证。 |
| 重复保护稳定 | 实时重复、历史重复均不重复写表。 |
| 图片策略清楚 | 成功保存、失败降级、路径长度都有处理。 |
| 回调线程安全 | 不在 SDK 回调线程写库写文件。 |
| 排除设备生效 | 配置排除只影响阶段 7 事件能力。 |
| 数据库零结构变更 | 不修改现有表结构，不新增阶段 7 表。 |
