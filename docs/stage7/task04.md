# 阶段 7 / 任务 04：抓拍图片保存与 raw payload

## 目标

实现抓拍图片保存和原始载荷记录策略。阶段 7 应把 ACS 事件中的图片保存到 `FaceEventLogging.SnapshotRootDirectory`，并将路径写入 `attendance_gate_v2.snapshot_path`。

## 目录规则

| 项目 | 规则 |
| --- | --- |
| 根目录 | `FaceEventLogging.SnapshotRootDirectory`，默认 `snapshots`。 |
| 相对路径 | 相对运行目录解析。 |
| 不存在 | 模块启动时自动创建。 |
| 创建失败 | 人脸事件模块启动失败或进入不可用状态。 |
| 文件分层 | 建议按 `yyyyMMdd/deviceId/` 分目录。 |

## 文件命名

建议格式：

```text
{eventTime:yyyyMMddHHmmssfff}_{deviceId}_{employeeId}_{eventId}.jpg
```

| 要求 | 说明 |
| --- | --- |
| 文件名安全 | 过滤员工编号中的非法路径字符。 |
| 可追踪 | 包含时间、设备、员工、事件 ID。 |
| 防覆盖 | 同名存在时可追加短后缀，但数据库 `id` 仍负责业务防重。 |

## 图片写入流程

| 步骤 | 动作 |
| --- | --- |
| 1 | 判断事件是否包含图片字节。 |
| 2 | 无图片则 `snapshot_path = NULL`，不影响入库。 |
| 3 | 校验图片大小和基础格式。 |
| 4 | 生成目录和文件名。 |
| 5 | 写入临时文件。 |
| 6 | 写入完成后原子移动到目标文件。 |
| 7 | 返回用于入库的路径。 |

## 路径保存

`snapshot_path` 长度为 `NVARCHAR(255)`。保存前必须检查长度：

| 场景 | 处理 |
| --- | --- |
| 路径长度 <= 255 | 正常写入。 |
| 路径长度 > 255 | 使用较短文件名或相对路径。 |
| 仍超过 255 | 不写图片路径，raw payload 记录原因。 |

优先保存相对运行目录的路径，便于发布包迁移。

## 文件写入失败

| 场景 | 处理 |
| --- | --- |
| 磁盘不可写 | 记录 `Error`，事件仍可入库但 `snapshot_path = NULL`。 |
| 图片字节为空 | 正常入库，路径为空。 |
| 图片格式未知 | 保存为 `.bin` 或不保存，按配置决定；默认不保存并记录。 |
| 同名冲突 | 追加短后缀。 |

抓拍保存失败不应阻塞事件入库，除非模块启动时根目录不可创建。

## raw payload

raw payload 应包含图片保存状态：

| 字段 | 说明 |
| --- | --- |
| `snapshotSaved` | 是否保存成功。 |
| `snapshotPath` | 保存路径。 |
| `snapshotError` | 保存失败原因。 |
| `pictureBytes` | 图片字节长度，不保存图片内容。 |

数据库 raw payload 不保存完整图片 Base64，避免流水表膨胀。日志是否记录完整字段由配置控制。

## 清理策略

阶段 7 不删除历史抓拍图片，不做图片归档。现场如需归档或清理，应作为运维策略或后续任务单独讨论。

## 不做的事

| 不做内容 | 原因 |
| --- | --- |
| 不把图片写入数据库 | 现有表只有路径字段，且避免表膨胀。 |
| 不修改 snapshot_path 长度 | 现有表结构零变更。 |
| 不实现图片归档 | 不属于阶段 7 核心入库链路。 |
| 不保存 AIOP 图片 | AIOP 属于后续阶段。 |

## 测试

| 测试 | 验证 |
| --- | --- |
| 目录自动创建 | 根目录不存在时可创建。 |
| 图片保存成功 | 文件存在且路径入库。 |
| 无图片事件 | 入库成功，路径为空。 |
| 路径超长 | 自动缩短或记录原因。 |
| 写文件失败 | 事件仍可入库，日志有错误。 |
| 文件名非法字符 | 被安全替换。 |
