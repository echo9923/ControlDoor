# 阶段 9 / 任务 06：阶段测试、联调与验收

## 目标

定义阶段 9 的单元测试、mock SDK 集成测试、配置测试、现场联调和通过标准，证明摄像头 AIOP 报警联动门禁常闭窗口可以在默认关闭、配置启用的前提下安全运行。

## 单元测试

| 测试类 | 覆盖内容 |
| --- | --- |
| `AiopAlarmEventRouterTests` | `0x4021` 识别、来源摄像头反查、未命中配置。 |
| `AiopVideoPayloadParserTests` | `NET_AIOP_VIDEO_HEAD`、JSON 长度、JPEG 签名、解析失败。 |
| `InterlockMappingResolverTests` | IP 优先、ID 兜底、DoorNos 默认、多目标解析。 |
| `CameraAlarmWindowManagerTests` | 首次窗口、重复报警不续期、新窗口开启。 |
| `DoorTargetStateManagerTests` | 多摄像头共享门、活动集合、恢复条件。 |
| `DoorControlTaskFactoryTests` | 常闭/恢复命令、优先级、门号、TaskKey。 |
| `RestoreRetryPolicyTests` | 恢复失败对可重试错误持续重试、不可重试错误转终态。 |

## Mock SDK 集成测试

| 场景 | 预期 |
| --- | --- |
| 未启用配置 | 收到 `0x4021` 不触发门控。 |
| 摄像头命中配置 | 投递常闭任务。 |
| JSON 解析失败 | 仍投递常闭任务。 |
| type 非短袖短裤 | 仍投递常闭任务。 |
| 窗口内重复报警 | 不刷新窗口，不重复常闭。 |
| 窗口结束 | 投递恢复任务。 |
| 一个摄像头多门 | 全部门目标常闭并恢复。 |
| 两摄像头共享门 | 最后一个窗口结束才恢复。 |
| 恢复失败 | 可重试错误持续重试，不可重试错误转终态，不阻塞 worker。 |
| 服务停止 | 活动门目标 best-effort 恢复。 |

## 配置测试

| 场景 | 验证 |
| --- | --- |
| `Enabled` 缺失 | 默认 false。 |
| `Mappings` 为空且启用 | 模块不可用并记录错误。 |
| `DoorNos` 缺失 | 默认 `[1]`。 |
| `WindowSeconds` 非法 | 回退默认值。 |
| 恢复失败可重试错误 | 持续重试直至成功，无最大次数。 |
| 恢复失败不可重试错误 | 转终态并记录 Error 日志。 |
| 摄像头 IP 和 ID 同时存在 | 优先 IP。 |
| 门禁 IP 和 ID 同时存在 | 优先 IP。 |

## 现场联调

真实联调必须显式开启，并使用测试摄像头、测试门禁和短窗口。

| 联调项 | 操作 | 通过标准 |
| --- | --- | --- |
| AIOP 回调 | 触发摄像头短衣短裤/工装报警。 | 服务收到 `0x4021`。 |
| 来源匹配 | 配置摄像头 IP。 | 日志显示命中 mapping。 |
| 常闭动作 | 报警触发门禁常闭。 | 门禁进入禁止入门/常闭状态。 |
| 窗口不续期 | 窗口内多次触发。 | 结束时间不刷新。 |
| 恢复动作 | 等待窗口结束。 | 门禁恢复普通受控状态。 |
| 共享门 | 两个摄像头映射同一门。 | 最后一个窗口结束才恢复。 |
| 停止清理 | 窗口内停止服务。 | 记录 best-effort 恢复结果。 |

## 安全检查

| 检查 | 标准 |
| --- | --- |
| 默认关闭 | 未配置启用时不影响首期能力。 |
| 不写数据库 | 阶段 9 默认不新增、不修改数据库。 |
| 不补偿旧窗口 | 设备离线后不按旧报警窗口补偿常闭。 |
| 恢复可见 | 恢复失败必须有 Error 日志。 |
| 人工处理入口 | 停止恢复失败日志包含门禁设备和门号。 |

## 阶段 9 通过标准

| 标准 | 说明 |
| --- | --- |
| AIOP 命中可触发 | `0x4021` 且摄像头命中配置即联动。 |
| 不按 type 过滤 | JSON type 不影响触发。 |
| 窗口语义正确 | 不续期、不重复常闭、新窗口可开启。 |
| 多目标正确 | 一个摄像头多个门、多个摄像头一个门都正确。 |
| 恢复可靠 | 窗口结束、恢复失败重试、服务停止恢复可验证。 |
| 设备通道化 | 常闭/恢复均通过 `DeviceSdkDispatcher`。 |
| 首期兼容 | 默认关闭时不影响阶段 1-8 能力和测试。 |
