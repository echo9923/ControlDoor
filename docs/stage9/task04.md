# 阶段 9 / 任务 04：摄像头窗口与门目标活动集合

## 目标

实现报警窗口状态管理。每个摄像头维护独立窗口，首次报警开启窗口；窗口内重复报警不刷新结束时间。每个门目标维护活动摄像头集合，只有最后一个摄像头窗口结束后才恢复门禁普通受控状态。

## 状态模型

建议内存状态：

| 模型 | 字段 |
| --- | --- |
| `CameraAlarmWindow` | `CameraKey`、`StartedAt`、`EndsAt`、`TriggeredCount`、`TargetKeys`。 |
| `DoorTargetActivity` | `TargetKey`、`DoorDeviceId`、`DoorNo`、`ActiveCameraKeys`、`AlwaysCloseSubmittedAt`、`RestoreSubmittedAt`。 |

阶段 9 默认不持久化窗口状态。服务重启时通过停止恢复策略降低门禁遗留风险。

## 摄像头窗口规则

| 场景 | 行为 |
| --- | --- |
| 摄像头无活动窗口 | 创建窗口，`EndsAt = now + WindowSeconds`。 |
| 摄像头已有活动窗口 | 不刷新 `EndsAt`，只增加计数并记录日志。 |
| 窗口到期 | 移除该摄像头窗口。 |
| 窗口到期后再次报警 | 开启新窗口。 |

示例：`WindowSeconds = 5`，第 1 秒报警开启到第 6 秒；第 2-5 秒重复报警不续期；第 6.1 秒报警开启新窗口。

## 门目标活动集合规则

| 场景 | 行为 |
| --- | --- |
| 摄像头窗口开始 | 把摄像头加入每个门目标的 `ActiveCameraKeys`。 |
| 门目标首次有活动摄像头 | 投递常闭任务。 |
| 门目标已有活动摄像头 | 不重复投递常闭，只记录活动集合变化。 |
| 某摄像头窗口结束 | 从对应门目标移除该摄像头。 |
| 门目标活动集合清空 | 投递恢复任务。 |

## 定时调度

窗口结束不应阻塞 worker。实现方式：

| 方式 | 规则 |
| --- | --- |
| 延迟调度器 | 为每个摄像头窗口创建到期任务。 |
| 到期校验 | 到期时再次检查窗口 `EndsAt`，避免旧任务误恢复。 |
| 合并 key | `CameraKey + WindowEndsAt`。 |
| 停止服务 | 取消未执行到期任务，进入停止恢复流程。 |

## 并发保护

| 场景 | 处理 |
| --- | --- |
| 同摄像头并发报警 | 对 `CameraKey` 加内存锁或串行化处理。 |
| 多摄像头共享门 | 对 `DoorTarget` 加锁更新活动集合。 |
| 常闭任务投递失败 | 保留活动集合，并记录失败；可按策略重试常闭或等待下一次报警。 |
| 恢复任务投递失败 | 进入任务 9.5 的恢复重试。 |

## 日志

| 事件 | 内容 |
| --- | --- |
| 窗口开始 | cameraKey、startsAt、endsAt、targetCount。 |
| 重复报警 | cameraKey、endsAt、triggeredCount。 |
| 窗口结束 | cameraKey、targetKeys。 |
| 门目标常闭 | targetKey、activeCameraKeys。 |
| 门目标等待恢复 | targetKey、remainingCameraKeys。 |
| 门目标恢复 | targetKey。 |

## 不做的事

| 不做内容 | 原因 |
| --- | --- |
| 不刷新窗口 | 用户已确认窗口内重复报警不续期。 |
| 不持久化窗口 | 阶段 9 默认内存状态。 |
| 不按 type 管理窗口 | 命中配置摄像头即联动。 |
| 不绕过延迟调度 | 窗口结束通过调度任务触发。 |

## 测试

| 测试 | 验证 |
| --- | --- |
| 首次报警 | 创建窗口并投递常闭。 |
| 重复报警 | 不刷新 `EndsAt`，不重复常闭。 |
| 窗口后报警 | 开启新窗口。 |
| 一个摄像头多门 | 多个门目标都进入活动。 |
| 多摄像头共享门 | 最后窗口结束才恢复。 |
| 旧到期任务 | 不误恢复新窗口。 |
