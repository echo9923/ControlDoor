# 阶段 3：海康 SDK/ISAPI 网关 — 实施目标与任务清单

## 概述

阶段 3 的目标是**封装海康 SDK 与 ISAPI 的全部底层调用**，向上层（阶段 4 设备生命周期、阶段 5 权限人员人脸同步、阶段 7 事件入库、阶段 9 AIOP 联动）提供统一、可测试、可 mock 的网关接口。

阶段 3 **不连接真实设备、不投递设备任务、不实现 gRPC 方法、不写数据库**。它只负责把海康 SDK 的 C API 和 ISAPI HTTP 请求封装成 C# 接口与实现，使后续阶段可以：

- 通过 `IHikvisionGateway` 接口调用登录/登出/布防/撤防/远程配置/人脸操作/门控/事件回调等能力；
- 在单元测试中用 mock 实现替换真实 SDK，不依赖物理设备；
- 统一处理 SDK 错误码、ISAPI 状态码、超时、重试和日志。

## 硬性边界

| 边界 | 要求 |
| --- | --- |
| SDK DLL 不提交 | `HCNetSDK.dll` 等海康 DLL 不进入 Git，仅通过 `DllPresenceHealthCheck` 在运行时检查存在性。 |
| 接口与实现分离 | 所有 SDK/ISAPI 调用必须通过 `IHikvisionGateway` 接口暴露，提供至少一个真实实现和一个 mock 实现。 |
| 不连接真实设备 | 阶段 3 的单元测试全部使用 mock，不发起真实网络连接。 |
| 错误码统一 | SDK 错误码、ISAPI HTTP 状态码统一映射为 `SdkError` 或 `DeviceGatewayException`，不向上层泄漏原生 `int` 错误码。 |
| 线程安全 | SDK 回调（如报警、人脸对比）来自外部线程，网关必须将回调数据复制到托管对象再向下传递，不在回调线程执行耗时操作。 |
| 不做业务逻辑 | 阶段 3 不包含设备生命周期管理、权限同步、事件入库、补偿重试等业务判断，只提供原子化的 SDK/ISAPI 调用能力。 |

## 任务 3.1：网关接口定义

### 目标

定义 `IHikvisionGateway` 接口和所有相关的请求/响应 DTO，形成阶段 3 及后续各阶段的**唯一底层契约**。任何阶段不得绕过此接口直接调用 `HCNetSDK` 的 `[DllImport]` 方法。

### 接口分组

| 分组 | 方法 | 说明 |
| --- | --- | --- |
| 连接管理 | `LoginAsync`、`LogoutAsync` | 设备登录与登出，返回/释放 UserID。 |
| 报警管理 | `SetAlarmAsync`、`CloseAlarmAsync` | 布防与撤防，传入 AlarmHandle 回调。 |
| 人员管理 | `AddPersonAsync`、`DeletePersonAsync`、`ModifyPersonAsync`、`QueryPersonAsync` | 人员 CRUD。 |
| 人脸管理 | `UploadFaceAsync`、`DeleteFaceAsync`、`QueryFaceAsync` | 人脸上传/删除/查询。 |
| 权限管理 | `SetPermissionAsync`、`QueryPermissionAsync` | 权限下发与查询。 |
| 门禁控制 | `ControlGatewayAsync` | 门禁常闭/恢复/远程开门。 |
| 抓拍 | `CaptureFaceAsync`、`CapturePictureAsync` | 人脸抓拍与通用抓拍。 |
| 事件 | `QueryEventRecordAsync` | 历史事件查询。 |
| ISAPI | `SendIsapiRequestAsync` | 通用 ISAPI HTTP 请求。 |
| 能力 | `GetDeviceCapabilitiesAsync`、`GetDeviceInfoAsync` | 设备能力集与基础信息。 |
| 工具 | `GetLastErrorCode`、`GetErrorMessage` | 错误码读取与中文描述。 |

### DTO 清单

| 类别 | DTO |
| --- | --- |
| 连接 | `LoginRequest`、`LoginResponse`、`LogoutRequest` |
| 报警 | `AlarmSetupRequest`、`AlarmCloseRequest`、`AlarmEventData`、`AlarmCallbackDelegate` |
| 人员 | `PersonInfo`、`AddPersonRequest`、`DeletePersonRequest`、`ModifyPersonRequest`、`QueryPersonRequest`、`QueryPersonResponse` |
| 人脸 | `FaceInfo`、`UploadFaceRequest`、`DeleteFaceRequest`、`QueryFaceRequest`、`QueryFaceResponse` |
| 权限 | `PermissionInfo`、`SetPermissionRequest`、`QueryPermissionRequest`、`QueryPermissionResponse` |
| 门控 | `GateControlRequest`、`GateControlResponse` |
| 抓拍 | `CaptureRequest`、`CaptureResponse`、`FaceCaptureResult` |
| 事件 | `EventQueryRequest`、`EventRecord`、`EventQueryResponse` |
| ISAPI | `IsapiRequest`、`IsapiResponse` |
| 能力 | `DeviceCapabilities`、`DeviceInfo` |
| 通用 | `SdkError`、`DeviceGatewayException` |

### 验收标准

- [ ] `IHikvisionGateway` 接口编译通过，所有方法签名可被 VS 智能提示发现。
- [ ] 每个方法都有 `<summary>` XML 文档注释，说明用途、参数和返回值。
- [ ] DTO 全部为 `class` 或 `struct`，使用 `System.Text.Json` 或 `Newtonsoft.Json` 可序列化。
- [ ] 不存在任何 `IntPtr`、`void*`、`uint` 等原生类型泄漏到接口或 DTO 中。

## 任务 3.2：SDK 基础封装（登录/登出/错误码）

### 目标

实现 `HikvisionSdkWrapper` 的基础部分：`LoginAsync`、`LogoutAsync`、`GetLastErrorCode`、`GetErrorMessage`，以及 `HCNetSDK` 的 `[DllImport]` 声明。

### 技术依据

- `设备网络SDK编程指南（明眸-以人为中心）/` 中关于 `NET_DVR_Login_V40`、`NET_DVR_Logout`、`NET_DVR_GetLastError` 的说明。
- SDK 调用约定：`__stdcall`，使用 `Encoding.UTF-8` 或 `Encoding.Default` 处理字符串。

### 关键实现点

1. **DllImport 声明**：在 `NativeMethods` 或 `HikvisionSdkNative` 内部类中集中声明所有 SDK 函数，使用 `[DllImport("HCNetSDK.dll")]`。
2. **登录**：
   - 调用 `NET_DVR_Login_V40`，传入 `NET_DVR_USER_LOGIN_INFO` 和 `NET_DVR_DEVICEINFO_V40`。
   - 支持 TCP/UDP 登录模式，默认 TCP。
   - 成功返回 `LoginResponse { UserId, DeviceInfo }`。
   - 失败抛出 `DeviceGatewayException`，内含 `SdkError { Code, Message }`。
3. **登出**：
   - 调用 `NET_DVR_Logout(userId)`。
   - 登出失败记录警告但不抛异常（设备可能已断连）。
4. **错误码**：
   - `GetLastErrorCode()` 调用 `NET_DVR_GetLastError` 返回 `int`。
   - `GetErrorMessage(code)` 调用 `NET_DVR_GetErrorMsg` 返回中文消息。
   - 提供静态方法 `SdkError.FromLastError()` 一次性获取错误码和消息。

### 验收标准

- [ ] `HikvisionSdkWrapper` 的 `LoginAsync` 和 `LogoutAsync` 在 mock SDK 下正常返回。
- [ ] `GetLastErrorCode` 在 mock 场景下返回可控错误码。
- [ ] 错误码 0 映射为 "成功"，常见错误码（如 1、7、17、152）均能返回正确中文描述。
- [ ] 单元测试覆盖：登录成功、登录失败（错误密码）、登出成功、登出失败不抛异常。

## 任务 3.3：SDK 报警布防/撤防封装

### 目标

封装 `NET_DVR_SetupAlarmChan_V41` / `NET_DVR_CloseAlarmChan`，将海康 C 风格回调转换为 C# 事件或 `IObservable<T>`。

### 技术依据

- `设备网络SDK编程指南（明眸-以人为中心）/` 中关于 `COMM_ALARM_ACS`（门禁事件）、`COMM_ALARM_FACE`（人脸对比事件）、`COMM_UPLOAD_AIOP_VIDEO`（AIOP 报警）的说明。
- 回调线程约束：回调内不得执行耗时操作，必须尽快复制数据后返回。

### 关键实现点

1. **布防**：
   - 调用 `NET_DVR_SetupAlarmChan_V41(userId, ref setupInfo)`。
   - 注册一个托管回调 `MSGCallBack`，将 `AlarmHandle` 返回给调用方。
2. **回调适配**：
   - 回调中根据 `dwCommand` 识别报警类型。
   - 将 `IntPtr` 数据 `Marshal.PtrToStructure` 为对应结构体，再复制到 `AlarmEventData` DTO。
   - 通过 `System.Reactive` 或 `Action<AlarmEventData>` 将事件传递给上层。
3. **撤防**：
   - 调用 `NET_DVR_CloseAlarmChan(alarmHandle)`。
   - 撤防后确保回调不再触发。

### 验收标准

- [ ] `SetAlarmAsync` 在 mock 下返回有效 `AlarmHandle`。
- [ ] `CloseAlarmAsync` 能正确关闭已建立的布防。
- [ ] Mock 可模拟 `COMM_ALARM_ACS` 事件回调，验证数据结构正确复制。
- [ ] 重复布防、重复撤防、未布防撤防等边界场景均有测试。

## 任务 3.4：SDK 远程配置封装（人员/人脸/权限）

### 目标

封装人员增删改查、人脸图片上传/删除/查询、权限下发等远程配置接口。

### 技术依据

- `设备网络SDK编程指南（明眸-以人为中心）/` 中关于 `NET_DVR_SetSDKCardInfo`、`NET_DVR_GetSDKCardInfo`、`NET_DVR_SetFacePic`、`NET_DVR_GetFacePic`、`NET_DVR_DelFacePic`、`NET_DVR_SetCardCfg` 等。
- ISAPI 路径 `/ISAPI/AccessControl/UserManagement/`、`/ISAPI/Intelligent/FDLib/` 也可作为替代或补充方案。

### 关键实现点

1. **人员 CRUD**：
   - 使用 `NET_DVR_STD_CARD_INFO` 或 ISAPI 实现人员增删改查。
   - 支持按 `employee_id`（工号）作为设备端唯一标识。
2. **人脸操作**：
   - 上传：调用人脸录入接口，传入 Base64 解码后的 JPEG 二进制数据。
   - 删除：按 `employee_id` + `card_no` 或人脸 ID 删除。
   - 查询：获取设备端已注册人脸列表。
   - 校验：上传前人脸图片大小、分辨率、格式预校验。
3. **权限下发**：
   - 按 `employee_id` 下发 `permission_code` 到指定门或门组。
   - 支持批量下发，设置合理的超时时间。

### 验收标准

- [ ] `AddPersonAsync` / `DeletePersonAsync` / `ModifyPersonAsync` / `QueryPersonAsync` 按接口契约实现。
- [ ] `UploadFaceAsync` 支持 JPEG 格式，自动校验图片大小不超过设备限制。
- [ ] `SetPermissionAsync` 能正确处理权限编号与门号的映射。
- [ ] 单元测试覆盖：成功、设备满容量、图片过大、格式不支持、人员已存在、权限冲突。

## 任务 3.6：ISAPI 请求封装

### 目标

封装 HTTP ISAPI 通用请求，支持 GET/POST/PUT/DELETE 方法，自动处理 Digest 认证和 JSON/XML 响应解析。

### 技术依据

- 海康 ISAPI 协议文档，基础路径 `/ISAPI/`。
- Digest 认证流程：`401 Unauthorized` → 解析 `WWW-Authenticate` → 生成 `Authorization` 头 → 重试。
- 使用 `HttpClient` 或 `HttpWebRequest`（.NET Framework 4.8 兼容）。

### 关键实现点

1. **认证**：
   - 首次请求收到 401 后自动执行 Digest 认证握手。
   - 支持缓存认证状态避免每次握手。
2. **请求方法**：
   - GET 用于查询、POST/PUT 用于修改、DELETE 用于删除。
3. **超时与重试**：
   - 默认超时 30 秒，可配置。
   - 网络错误自动重试 1 次（幂等请求）。
4. **响应处理**：
   - JSON 响应直接反序列化。
   - XML 响应转为 `XDocument` 或提取关键字段。

### 验收标准

- [ ] `SendIsapiRequestAsync` 支持 GET/POST/PUT/DELETE。
- [ ] Digest 认证流程在 mock HTTP server 下验证通过。
- [ ] 超时场景正确处理，不无限等待。
- [ ] 单元测试使用 `HttpMessageHandler` mock 覆盖认证成功、认证失败、超时、404、500。

## 后续阶段衔接

阶段 3 完成后，后续阶段的使用方式为：

```csharp
// 阶段 4 设备生命周期示例
var loginResult = await _gateway.LoginAsync(new LoginRequest
{
    IpAddress = device.IpAddress,
    Port = device.Port,
    UserName = device.Username,
    Password = device.Password
});
// loginResult.UserId 用于后续所有 SDK 操作

// 阶段 5 人员同步示例
await _gateway.AddPersonAsync(new AddPersonRequest
{
    UserId = loginResult.UserId,
    Person = new PersonInfo { EmployeeId = "10001", Name = "张三" }
});

// 阶段 7 事件回调示例
_gateway.OnAlarmEvent += (sender, data) =>
{
    // data 已是托管对象，可安全投递到后台队列
    _eventQueue.Enqueue(data);
};
```
