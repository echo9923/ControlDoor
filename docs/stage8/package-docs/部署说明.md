# ControlDoor 部署说明

## 发布包位置

现场发布包固定使用：

```text
门禁publish\ServicePackage
```

该目录必须包含：

```text
ControlDoor.exe
ControlDoor.exe.config
Configuration\appsettings.json
Configuration\devices.json
tools\service\install-service.ps1
tools\service\start-service.ps1
tools\service\stop-service.ps1
tools\service\uninstall-service.ps1
logs\
snapshots\
docs\
```

## 安装前步骤

1. 确认 Hikvision SDK DLL 与进程位数一致。
2. 确认 `HCNetSDK.dll` 位于 exe 同级或 `sdk\Hikvision\HCNetSDK.dll`。
3. 确认 `SqlServerTypes` 位于 exe 同级或 `sdk\SqlServerTypes`。
4. 修改 `Configuration\appsettings.json` 中的数据库、端口、日志和 `Devices.FilePath` 等配置。
5. 修改 `Configuration\devices.json` 中的设备清单；每台设备必须包含 `deviceId`、`name`、`types`、`ipAddress`、`port`、`username`、`password`、`enabled`、`remark`。
6. 正式部署建议保持 `Devices.Items` 为空，通过 `Configuration\devices.json` 或设备管理 gRPC 维护设备。若使用内联 `Devices.Items`，`AddDevice`/`DeleteDevice` 只更新当前运行时缓存，不写文件。
7. 执行 `ControlDoor.exe --validate-config`，必须通过后再安装服务。

## 服务脚本

```powershell
powershell -ExecutionPolicy Bypass -File tools\service\install-service.ps1 -PackageRoot .
powershell -ExecutionPolicy Bypass -File tools\service\start-service.ps1 -PackageRoot .
powershell -ExecutionPolicy Bypass -File tools\service\stop-service.ps1
powershell -ExecutionPolicy Bypass -File tools\service\uninstall-service.ps1
```

服务名和显示名固定为 `ControlDoor`。卸载脚本只移除 Windows Service，不删除配置、日志、抓拍和数据库数据。

## 回滚

1. 执行 `tools\service\stop-service.ps1` 停止当前服务。
2. 备份当前 `ServicePackage`、`Configuration\appsettings.json` 和 `Configuration\devices.json`。
3. 恢复上一版 `ServicePackage` 和上一版配置。
4. 执行 `ControlDoor.exe --validate-config`。
5. 执行 `tools\service\start-service.ps1` 启动旧版本。
