# ControlDoor 运行前检查

## 自动检查

在发布包根目录执行：

```powershell
.\ControlDoor.exe --validate-config
```

验证模式只做运行前检查，不启动 gRPC 服务，不启动设备 worker，不下发任何设备操作。

## 必须通过项

| 项目 | 通过标准 |
| --- | --- |
| 配置文件 | `Configuration\appsettings.json` 存在，JSON 可解析。 |
| gRPC 端口 | `Service.GrpcListenPort` 在 1-65535，且未被占用。 |
| 数据库 | 连接成功，`dbo.system_users`、`dbo.attendance_gate_v2`、`dbo.device_operation_retry_states` 可读。 |
| 设备清单 | `Configuration\devices.json` 存在且可解析；设备条目包含合法 `types`。 |
| 阶段 6 表 | 启用离线补偿时 `dbo.device_operation_retry_states` 可读。 |
| 阶段 7 表 | 启用人脸事件时 `dbo.attendance_gate_v2` 可读。 |
| 日志目录 | `Logging.LogDirectory` 可创建且可写。 |
| SDK 日志目录 | `HikvisionSdk.SdkLogDirectory` 可创建；`RequireSdkLog=true` 时必须可写。 |
| 抓拍目录 | `FaceEventLogging.SnapshotRootDirectory` 可创建且可写。 |
| SDK DLL | 找到 `HCNetSDK.dll`。 |
| SQL Server Types | 找到 `SqlServerTypes`。 |

## 人工确认项

| 项目 | 确认 |
| --- | --- |
| SDK 位数 | 与服务进程平台一致。 |
| 设备白名单 | 仅包含允许真实联调的设备 ID/IP。 |
| 测试员工 | 使用专用测试员工、权限和人脸图片。 |
| 日志策略 | 确认 `Logging.LogDirectory`、`RetentionDays`、`MinimumLevel` 和慢操作阈值符合现场要求。 |
| 密钥 | 仓库模板中不得出现真实数据库密码和正式 API Key。 |

## 失败处理

validate 失败时不要安装或启动服务。先根据失败项处理配置、端口、数据库、目录权限或 SDK DLL，再重新执行 `ControlDoor.exe --validate-config`。
