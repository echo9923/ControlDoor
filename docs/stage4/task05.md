# 阶段 4 / 任务 05：布防编排与撤防策略

## 目标

把阶段 3.7 的报警布防基础封装接入设备生命周期。登录成功后按配置投递布防任务；删除、断开、重连和服务停止前执行撤防；运行时维护 AlarmHandle 和反查索引。

## 布防触发点

| 触发点 | 行为 |
| --- | --- |
| 启动登录成功 | 如 `Alarm.Enabled=true`，投递布防任务。 |
| 手动重连成功 | 登录成功后重新布防。 |
| 自动重连成功 | 登录成功后重新布防。 |
| AddDevice 且 connectNow 成功 | 登录成功后按配置布防。 |

## 布防任务

| 字段 | 说明 |
| --- | --- |
| `OperationName` | `SetupAlarm`。 |
| `Priority` | `High`。 |
| `Input` | DeviceId、SdkUserId、布防参数。 |
| `Output` | AlarmHandle、布防时间、错误信息。 |

## 布防流程

| 步骤 | 动作 | 失败处理 |
| --- | --- | --- |
| 1 | 检查设备在线且 UserID 有效。 | 不在线返回 `DEVICE_ERROR`。 |
| 2 | 检查是否已有 AlarmHandle。 | 已布防则返回当前句柄。 |
| 3 | 检查全局报警回调已注册。 | 未注册则返回 `SDK_ERROR`。 |
| 4 | 调用 `SetupAlarmChannel`。 | 失败记录错误，设备仍保持在线。 |
| 5 | 保存 AlarmHandle 到运行时。 | 索引失败时尝试撤防。 |
| 6 | 建立 AlarmHandle 反查索引。 | 用于回调来源定位和停止清理。 |

## 撤防触发点

| 触发点 | 行为 |
| --- | --- |
| DeleteDevice | 删除前撤防。 |
| DisconnectDevice | 登出前撤防。 |
| ReconnectDevice(force=true) | 重新登录前撤防。 |
| 自动重连前旧会话清理 | 旧 UserID 仍存在时先撤防。 |
| 服务停止 | 对所有已布防设备 best-effort 撤防。 |

## 撤防流程

| 步骤 | 动作 |
| --- | --- |
| 1 | 读取运行时 AlarmHandle。 |
| 2 | 调用 `CloseAlarmChannel`。 |
| 3 | 无论 SDK 成功失败，都清理本地 AlarmHandle。 |
| 4 | 清理 AlarmHandle 反查索引。 |
| 5 | 记录撤防结果和原因。 |

## 失败策略

| 失败 | 策略 |
| --- | --- |
| 布防失败 | 设备仍可在线；事件功能标记不可用；按配置安排布防重试。 |
| 撤防失败 | 记录 warning，继续登出/删除流程；本地句柄清理。 |
| 回调未注册 | 布防失败，提示先注册回调。 |
| AlarmHandle 索引冲突 | 尝试撤防新句柄并返回内部错误。 |

## 不做的事

| 不做内容 | 原因 |
| --- | --- |
| 不解析回调 payload | 阶段 7 负责。 |
| 不写事件表 | 阶段 7 负责。 |
| 不触发摄像头联动 | 后续阶段负责。 |
| 不写数据库保存 AlarmHandle | AlarmHandle 是运行时资源，不持久化。 |

## 测试

| 测试 | 验证 |
| --- | --- |
| 登录后布防成功 | AlarmHandle 写入运行时和索引。 |
| 重复布防 | 不重复调用 SDK。 |
| 布防失败 | 设备保持在线，事件状态为失败。 |
| 索引失败 | 尝试撤防新句柄。 |
| 删除前撤防 | 撤防发生在登出和删除数据库前。 |
| 撤防失败 | 本地索引仍清理，流程继续。 |
| 服务停止 | 对全部已布防设备执行 best-effort 撤防。 |
