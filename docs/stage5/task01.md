# 阶段 5 / 任务 01：阶段边界与任务总览

## 目标

阶段 5 实现权限、人员、人脸同步相关 gRPC 方法：

- `SyncPermissions`
- `SyncPersons`
- `DeleteFaces`
- `DeletePersons`
- `GetFaces`
- `CaptureFaceStream`
- `GetEnrollmentStatus`

阶段 5 负责解析请求、选择目标在线设备、通过设备固定执行通道调用阶段 3/4 提供的 SDK/ISAPI/远程配置能力，并返回员工维度、设备维度的结果明细。

## 硬性边界

| 边界 | 要求 |
| --- | --- |
| gRPC 完全兼容 | 方法名、方法类型、请求容器、字段别名、响应字段和错误码保持不变。 |
| 批量上限 | 权限、人员、人脸、删除类接口最大 500 条，超出返回 `BATCH_TOO_LARGE`。 |
| 设备操作通道化 | 所有设备写入、查询、采集操作必须进入 `DeviceSdkDispatcher`。 |
| 人员表零结构变更 | 阶段 5 可更新 `system_users` 既有门禁同步字段，不修改表结构。 |
| 补偿只产出意图 | 阶段 5 可识别离线或可重试失败并调用补偿记录接口；补偿扫描、重试、终态和清理由阶段 6 详细实现。 |
| 不做事件入库 | ACS 事件实时入库和离线事件上传补偿属于阶段 7。 |
| 不做摄像头联动 | AIOP 联动属于后续阶段。 |

## 通用处理流程

| 步骤 | 动作 |
| --- | --- |
| 1 | 读取 requestId，解析 JSON 字符串。 |
| 2 | 识别请求容器和字段别名。 |
| 3 | 校验批量上限和必填字段。 |
| 4 | 解析为标准业务命令列表。 |
| 5 | 获取目标在线设备快照。 |
| 6 | 为每个设备创建设备任务。 |
| 7 | 汇总成功、失败、跳过、排队结果。 |
| 8 | 必要时更新 `system_users` 既有同步字段。 |
| 9 | 返回统一 JSON 响应。 |

## 目标设备选择

| 场景 | 规则 |
| --- | --- |
| 在线设备 | 可立即投递设备任务。 |
| 离线设备 | 不直接调用 SDK，产出补偿意图。 |
| 停用设备 | 不参与同步，计为 skipped。 |
| 配置非法设备 | 不参与同步，计为 failed 或 skipped，按接口语义返回。 |

## 通用响应

所有 Unary 方法必须包含：

```json
{
  "requestId": "请求ID",
  "success": true,
  "code": "OK",
  "message": "处理结果说明",
  "errors": [],
  "errorDetails": []
}
```

## 不做的事

| 不做内容 | 原因 |
| --- | --- |
| 不实现补偿扫描和重试 | 阶段 6 负责。 |
| 不解析 ACS 回调入库 | 阶段 7 负责。 |
| 不修改人员表结构 | 现有表零结构变更。 |
| 不绕过设备通道 | 保持同设备 SDK 串行。 |
