# 阶段 5 / 任务 05：GetFaces

## 目标

实现 `/permission.PermissionSyncService/GetFaces`，按员工编号查询设备端人脸信息，返回员工/设备维度结果。该接口只查询，不修改数据库，不产生补偿重试。

## 请求格式

请求格式同 `DeleteFaces`：

- 字符串数组
- 对象数组
- `items`
- `records`
- 单个对象

员工编号字段别名：`employee_id`、`employeeId`、`employee_no`、`employeeNo`。

## 执行流程

| 步骤 | 动作 |
| --- | --- |
| 1 | 解析员工编号列表。 |
| 2 | 校验批量上限。 |
| 3 | 获取在线设备列表。 |
| 4 | 对每个员工/设备投递查询人脸任务。 |
| 5 | 汇总设备成功、设备失败、离线跳过结果。 |
| 6 | 返回 `items` 明细。 |

## 响应字段

| 字段 | 说明 |
| --- | --- |
| `total` | 员工总数。 |
| `succeeded` | 成功数量。 |
| `failed` | 失败数量。 |
| `queued` | 待补齐数量，查询接口不写补偿，可用于表达离线未查。 |
| `targetDevices` | 目标设备数量。 |
| `items` | 查询结果。 |

`items` 建议字段：

| 字段 | 说明 |
| --- | --- |
| `employeeId` | 员工编号。 |
| `success` | 是否成功。 |
| `devices` | 设备维度结果。 |
| `faceImageBase64` | 如设备返回人脸图片，可返回。 |
| `rawResponse` | 设备原始响应，可按配置控制是否返回。 |
| `error` | 错误摘要。 |

## 设备端实现边界

| 能力 | 说明 |
| --- | --- |
| ISAPI 查询 | 可通过阶段 3.5 透传。 |
| 远程配置查询 | 可通过阶段 3.6 查询会话。 |
| 响应解析 | 业务层解析员工是否有人脸、图片内容或设备状态。 |
| 原始响应 | 是否完整返回由配置控制，避免响应过大。 |

## 不做的事

| 不做内容 | 原因 |
| --- | --- |
| 不写补偿表 | 查询不是写操作。 |
| 不修改人员表 | 查询只读。 |
| 不自动下发缺失人脸 | 下发由 `SyncPersons` 或阶段 6 补偿处理。 |

## 测试

| 测试 | 验证 |
| --- | --- |
| 多员工多设备 | 返回设备维度明细。 |
| 设备离线 | 查询跳过或 failed，接口不写补偿。 |
| 设备无此人脸 | 返回成功但标明不存在。 |
| 设备错误 | 返回设备错误明细。 |
| 原始响应过大 | 按配置截断或不返回。 |
